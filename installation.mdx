## Installation Methods

<Tabs>
  <Tab title="Automated Setup (Recommended)">
    ### Using the Initialization Script
    
    The easiest way to get started is using the provided `init_project.sh` script:
    
    <Steps>
      <Step title="Download the Script">
        Ensure you have the `init_project.sh` file in your project root:
        
        ```bash
        ls -la init_project.sh
        chmod +x init_project.sh
        ```
      </Step>
      
      <Step title="Run the Script">
        Execute the initialization script:
        
        ```bash
        ./init_project.sh
        ```
        
        The script will:
        - Verify Python installation
        - Create project directory structure
        - Set up a virtual environment
        - Install base dependencies
        - Optionally install development dependencies
      </Step>
      
      <Step title="Respond to Prompts">
        When asked:
        
        ```
        ¿Instalar dependencias de desarrollo también? (s/n):
        ```
        
        - Type `s` or `y` for development setup (includes testing, linting tools)
        - Type `n` for minimal production setup
      </Step>
    </Steps>
    
    <Note>
      The script is designed for Linux/Mac. Windows users should follow the Manual Setup instructions below.
    </Note>
  </Tab>
  
  <Tab title="Manual Setup">
    ### Step-by-Step Manual Installation
    
    <Steps>
      <Step title="Verify Python Installation">
        Check that Python 3.10+ is installed:
        
        ```bash
        python3 --version
        # Should output: Python 3.10.x or higher
        ```
        
        If not installed, download from [python.org](https://www.python.org/downloads/)
      </Step>
      
      <Step title="Create Virtual Environment">
        Create an isolated Python environment:
        
        **Linux/Mac:**
        ```bash
        python3 -m venv venv
        source venv/bin/activate
        ```
        
        **Windows:**
        ```bash
        python -m venv venv
        venv\Scripts\activate
        ```
        
        Your prompt should now show `(venv)` prefix.
      </Step>
      
      <Step title="Upgrade pip">
        Ensure you have the latest pip version:
        
        ```bash
        pip install --upgrade pip
        ```
      </Step>
      
      <Step title="Install Dependencies">
        Install based on your environment needs:
        
        <CodeGroup>
        ```bash Production
        # Base + Production dependencies
        pip install -r requirements/base.txt
        pip install -r requirements/production.txt
        ```
        
        ```bash Development
        # Base + Development dependencies
        pip install -r requirements/base.txt
        pip install -r requirements/development.txt
        ```
        
        ```bash Testing
        # Base + Testing dependencies
        pip install -r requirements/base.txt
        pip install -r requirements/testing.txt
        ```
        </CodeGroup>
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Database Setup

### PostgreSQL Installation

<Tabs>
  <Tab title="macOS">
    Using Homebrew:
    
    ```bash
    brew install postgresql@14
    brew services start postgresql@14
    ```
  </Tab>
  
  <Tab title="Ubuntu/Debian">
    Using apt:
    
    ```bash
    sudo apt update
    sudo apt install postgresql postgresql-contrib
    sudo systemctl start postgresql
    sudo systemctl enable postgresql
    ```
  </Tab>
  
  <Tab title="Windows">
    1. Download the installer from [postgresql.org](https://www.postgresql.org/download/windows/)
    2. Run the installer and follow the setup wizard
    3. Remember the password you set for the `postgres` user
  </Tab>
  
  <Tab title="Docker">
    Using Docker Compose:
    
    ```yaml docker-compose.yml
    version: '3.8'
    services:
      postgres:
        image: postgres:14
        environment:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: softbee_local
        ports:
          - "5432:5432"
        volumes:
          - postgres_data:/var/lib/postgresql/data
    
    volumes:
      postgres_data:
    ```
    
    Start the database:
    ```bash
    docker-compose up -d
    ```
  </Tab>
</Tabs>

### Create Database

Create the database for your environment:

```bash
# Connect to PostgreSQL
psql -U postgres

# Create database
CREATE DATABASE softbee_local;

# Create user (optional)
CREATE USER softbee_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE softbee_local TO softbee_user;

# Exit
\q
```

<Note>
  For different environments, create separate databases:
  - `softbee_local` - Local development
  - `softbee_dev` - Development server
  - `softbee_test` - Testing
  - `softbee_prod` - Production
</Note>

### Configuration Options

<ParamField path="FLASK_ENV" type="string" default="local">
  Environment mode: `local`, `development`, `production`, or `testing`
</ParamField>

<ParamField path="SECRET_KEY" type="string" required>
  Flask secret key for session management
  
  **Generate a secure key:**
  ```bash
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
</ParamField>

<ParamField path="JWT_KEY" type="string" required>
  Secret key for JWT token signing
  
  **Generate a secure key:**
  ```bash
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
</ParamField>

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL connection string
  
  **Format:**
  ```
  postgresql://username:password@host:port/database
  ```
  
  **Example:**
  ```
  postgresql://postgres:postgres@localhost:5432/softbee_local
  ```
</ParamField>

<ParamField path="ALGORITHM" type="string" default="HS256">
  JWT signing algorithm
</ParamField>

<ParamField path="EXPIRES_TOKEN_SESSION" type="integer" default="1440">
  Access token expiration in minutes (default: 24 hours)
</ParamField>

<ParamField path="EXPIRES_TOKEN_EMAIL" type="integer" default="30">
  Email verification token expiration in minutes
</ParamField>

<ParamField path="SMTP_HOST" type="string">
  SMTP server hostname (e.g., `smtp.gmail.com`)
</ParamField>

<ParamField path="SMTP_PORT" type="integer" default="587">
  SMTP server port
</ParamField>

<ParamField path="SMTP_USER" type="string">
  SMTP username/email for sending emails
</ParamField>

<ParamField path="SMTP_PASSWORD" type="string">
  SMTP password or app-specific password
</ParamField>

<ParamField path="FRONTEND_URL" type="string" default="http://localhost:3000">
  Frontend application URL for CORS and email links
</ParamField>

<ParamField path="BASE_URL" type="string" default="http://localhost:5000">
  Backend API base URL
</ParamField>

## Database Migrations

### Initialize Migrations

Set up Flask-Migrate for database versioning:

```bash
# Activate virtual environment
source venv/bin/activate  # Windows: venv\Scripts\activate

# Initialize migration directory (first time only)
flask db init
```

This creates a `migrations/` directory to track database changes.

### Create and Apply Migrations

<Steps>
  <Step title="Generate Migration">
    Create a migration script from model changes:
    
    ```bash
    flask db migrate -m "Initial migration"
    ```
    
    This analyzes your SQLAlchemy models and generates migration scripts.
  </Step>
  
  <Step title="Review Migration">
    Check the generated migration file in `migrations/versions/`:
    
    ```bash
    ls migrations/versions/
    ```
    
    Review the `upgrade()` and `downgrade()` functions to ensure correctness.
  </Step>
  
  <Step title="Apply Migration">
    Apply the migration to your database:
    
    ```bash
    flask db upgrade
    ```
    
    This executes the SQL commands to update your database schema.
  </Step>
</Steps>

### Common Migration Commands

```bash
# Show current migration status
flask db current

# Show migration history
flask db history

# Upgrade to latest
flask db upgrade

# Upgrade to specific version
flask db upgrade <revision>

# Downgrade one version
flask db downgrade

# Downgrade to specific version
flask db downgrade <revision>
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Python version issues" icon="python">
    If you get a Python version error:
    
    ```bash
    # Check your Python version
    python3 --version
    
    # Use a specific Python version
    python3.10 -m venv venv
    
    # Or install Python 3.10+ from python.org
    ```
  </Accordion>
  
  <Accordion title="PostgreSQL connection failed" icon="database">
    Common PostgreSQL connection issues:
    
    1. **PostgreSQL not running:**
       ```bash
       # Check status
       sudo systemctl status postgresql  # Linux
       brew services list                # macOS
       
       # Start service
       sudo systemctl start postgresql   # Linux
       brew services start postgresql    # macOS
       ```
    
    2. **Wrong credentials:**
       - Verify username and password in `.env`
       - Test connection: `psql -U postgres -h localhost`
    
    3. **Database doesn't exist:**
       ```bash
       psql -U postgres
       CREATE DATABASE softbee_local;
       ```
  </Accordion>
  
  <Accordion title="pip install fails" icon="box">
    If dependency installation fails:
    
    ```bash
    # Upgrade pip first
    pip install --upgrade pip setuptools wheel
    
    # Install with verbose output
    pip install -r requirements/base.txt -v
    
    # For psycopg2 issues on Mac:
    brew install postgresql
    pip install psycopg2-binary
    
    # For psycopg2 issues on Linux:
    sudo apt-get install libpq-dev python3-dev
    pip install psycopg2-binary
    ```
  </Accordion>
  
  <Accordion title="Migration errors" icon="database">
    If migrations fail:
    
    ```bash
    # Remove migrations directory
    rm -rf migrations/
    
    # Reinitialize
    flask db init
    flask db migrate -m "Initial migration"
    flask db upgrade
    
    # If still failing, check:
    # 1. Database exists and is accessible
    # 2. DATABASE_URL in .env is correct
    # 3. Models are imported in db.py
    ```
  </Accordion>
  
  <Accordion title="Virtual environment activation" icon="terminal">
    Platform-specific activation:
    
    **Linux/Mac (bash/zsh):**
    ```bash
    source venv/bin/activate
    ```
    
    **Linux/Mac (fish):**
    ```bash
    source venv/bin/activate.fish
    ```
    
    **Windows (CMD):**
    ```bash
    venv\Scripts\activate.bat
    ```
    
    **Windows (PowerShell):**
    ```bash
    venv\Scripts\Activate.ps1
    ```
    
    If PowerShell blocks scripts:
    ```powershell
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
    ```
  </Accordion>
  
  <Accordion title="Import errors" icon="code">
    If you see module import errors:
    
    ```bash
    # Ensure virtual environment is activated
    which python  # Should point to venv/bin/python
    
    # Reinstall dependencies
    pip install -r requirements/base.txt
    
    # Check if package is installed
    pip list | grep flask
    
    # Set PYTHONPATH if needed
    export PYTHONPATH="${PYTHONPATH}:$(pwd)"
    ```
  </Accordion>
</AccordionGroup>
